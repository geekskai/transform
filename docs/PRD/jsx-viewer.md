# PRD: Online JSX Viewer & Instant Previewer

**Keywords:** JSX Viewer, React Playground Online, JSX Formatter, JSX to HTML, Live React Preview.

## 1. Product Overview

### 1.1 Objective

To provide a lightweight, browser-based utility that allows developers to **paste, format, and instantly preview** JSX/React snippets without setting up a local development environment (Node.js, Vite, etc.).

### 1.2 Target Audience

- **React Developers:** Quick debugging or code sharing.
- **Students/Learners:** Understanding how JSX renders into the DOM.
- **Content Creators:** Generating clean, formatted code screenshots or snippets for blogs.
- **AI Users:** Quickly validating UI components generated by LLMs (like ChatGPT/Claude).

---

## 2. Core Features (Functional Requirements)

### 2.1 Smart Code Editor

- **Syntax Highlighting:** Full support for JSX, TSX, and ES6+ syntax.
- **Auto-Formatting:** Integrated with **Prettier** to "Beautify" messy code with one click.
- **Error Linting:** Real-time red underlines for syntax errors (missing brackets, unclosed tags).

### 2.2 Live Instant Preview

- **Real-time Rendering:** As the user types, the right-side panel renders the component.
- **Support for Tailwind CSS:** Enable a toggle to include Tailwind CDN so users can preview styled components.
- **Lucide-React / Icon Support:** Built-in support for common icon libraries often used in AI-generated code.

### 2.3 Transformation & Export

- **JSX to JS:** Show the compiled `React.createElement` output (powered by Babel).
- **Copy to Clipboard:** One-click copy for both raw and formatted code.
- **Download as Component:** Export the code as a `.jsx` or `.tsx` file.

## 4. Technical Specifications

| Feature         | Technology Recommendation                                            |
| --------------- | -------------------------------------------------------------------- |
| **Editor**      | **Monaco Editor** (the engine behind VS Code) or **CodeMirror 6**.   |
| **Compiler**    | **@babel/standalone** for browser-side JSX transpilation.            |
| **Formatting**  | **Prettier/standalone** for client-side code cleaning.               |
| **Sandboxing**  | Use `<iframe>` with `srcdoc` to isolate the preview and prevent XSS. |
| **Performance** | Debounced rendering (e.g., 300ms) to ensure smooth typing.           |
